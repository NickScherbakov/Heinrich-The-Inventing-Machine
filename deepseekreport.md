## Краткое описание проекта

Heinrich - это открытая система искусственного интеллекта для решения изобретательских задач на основе методологии TRIZ (Теория решения изобретательских задач). Проект назван в честь Генриха Альтшуллера, создателя TRIZ, и представляет собой AI-движок, который сочетает классическую методологию TRIZ с современными большими языковыми моделями.

### Ключевые особенности:
- **Системный пайплайн TRIZ:** 7-модульный рабочий процесс, следующий классической методологии TRIZ
- **Интерпретируемое рассуждение:** Каждый шаг логируется и отслеживается до принципов TRIZ
- **База знаний:** Полные 39 параметров, 40 принципов, матрица противоречий
- **Мультиязычная поддержка:** Полная документация на английском, китайском и русском языках
- **Этичный AI:** Прозрачная персона с руководствами по безопасности

### Основные модули

1. **Problem Parser** (`problem_parser.py`) - Извлекает сущность из неструктурированных описаний проблем
2. **Contradiction Identifier** - Отображает на 39 параметров TRIZ и идентифицирует противоречия
3. **Principle Selector** - Выбирает релевантные принципы из матрицы противоречий
4. **Effects Lookup** - Ссылается на научные эффекты и паттерны эволюции
5. **Concept Generator** - Создает концепции решений с отслеживаемостью TRIZ
6. **Adaptation Planner** - Контекстуализирует решения для реальных ограничений
7. **Report Builder** - Генерирует структурированные, отслеживаемые отчеты о решениях

## Рабочий процесс TRIZ

Проект реализует 7-шаговый пайплайн TRIZ:

```
Вход проблемы
     ↓
[Анализатор проблемы] → Нормализованное описание проблемы
     ↓
[Идентификатор противоречий] → Технические/физические противоречия
     ↓
[Селектор принципов] → Релевантные принципы TRIZ (1-40)
     ↓
[Поиск эффектов] → Научные эффекты и паттерны эволюции
     ↓
[Генератор концепций] → Концепции решений с обоснованием
     ↓
[Планировщик адаптации] → Контекстно-зависимые рекомендации
     ↓
[Сборщик отчетов] → Структурированный отчет о решении
```

```python
class HeinrichCLI:
    """Command-line interface for the Heinrich TRIZ Engine."""
    
    def run_interactive_mode(self):
        """Run the interactive problem-solving mode."""
        self.print_header()
        self.print_persona_greeting()
        
        # Step 1: Problem Input
        self.step_problem_input()
        
        # Step 2: Problem Analysis
        self.step_problem_analysis()
        
        # Step 3: Contradiction Identification
        self.step_contradiction_identification()
        
        # Step 4: Principle Selection
        self.step_principle_selection()
        
        # Step 5: Effects Integration
        self.step_effects_integration()
        
        # Step 6: Concept Generation
        self.step_concept_generation()
        
        # Step 7: Context Adaptation
        self.step_context_adaptation()
        
        # Step 8: Report Generation
        self.step_report_generation()
```

## Установка и использование

### Установка из исходного кода

```bash
git clone https://github.com/NickScherbakov/Heinrich-The-Inventing-Machine.git
cd Heinrich-The-Inventing-Machine
pip install -r requirements.txt
pip install -e .
```

### Быстрый пример использования

```python
from problem_parser import ProblemParser

parser = ProblemParser()
problem = "We need to make a car faster, but increasing engine power makes it consume more fuel."
result = parser.parse(problem)

print(f"Technical System: {result.technical_system}")
print(f"Desired Improvement: {result.desired_improvement}")
print(f"Undesired Consequence: {result.undesired_consequence}")
```

### Запуск CLI

```bash
# Интерактивный режим решения проблем
python3 heinrich_cli.py interactive
```
